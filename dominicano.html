Summary

Dominican√≥ is an application that allows you to play, learn or train in the Dominican domino. This application can be downloaded to your cell phone and it will give the feeling that you are playing domino just as you would in real life. This thanks to augmented reality technologies, thus using the camera of the user's mobile device, to create a gaming environment wherever they are. To achieve this, the application needs to be calibrated before starting to play, this is achieved simply by looking for a flat surface, which in effect the application will identify as a possible objective to position the table, once the device screen is touched the application will generate the table in the foreground in front of the midpoint of the screen, and finally the user will be able to train against the artificial intelligence programmed in the application or simply play for entertainment purposes. This application seeks to simulate the game of domino as we know it, which means that the interface complies with the way you grab chips with your hands and position them on the table.

The entirety of this project is made by one person, myself.

Description

This augmented reality project consists of a Mobile platform, a web page that implements a multiplayer version and allows "Crossplay" between one platform and another, and finally a server made in NodeJs, Typescript and SocketIO technologies.

The Web distribution works in conjunction with a server which implements the game logic of the CPU or artificial player. This logic uses a graphing algorithm to select the chips that the AI or CPU will play. 

The algorithm can be summarized in three parts:
1. The representation and / or transformation of the chips.
2. Identification of the possible paths in which a new tile in the game can enable
3. Decision making



Representation

For this representation use a graph algorithm to transform the tabs into adjacency lists.
For this, each number must be represented as a node, and each relationship between one token and another will be represented as an edge.
