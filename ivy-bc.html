Summary

In colaboration with a team of four lawyers and myself we created a full functional Web Application that implements blockchain technologies.

We used the classic Software Development Life Cycle (SDLC) to design this application called Ivy. Identifying and attacking a problematic that is facing owners of real state in Dominican Republic, we in collaboration with RI (Registro Inmobiliario) will be presenting this new idea of a faster and safer way to manage the property of each individual.

Description

Ivy is a fast and easy to use real state exchange application, that uses Elliptic Curve Cryptography standard, specifically the same finite curve as Bitcoin (secp256k1). The reason this was conceived was to speed up the process in which our peers and our field survey reflected as slow. Actually people have to wait more time than responsibly should. For this reason, we implemented a three-layer consensus between each legal institution that needs to approve the creation of each transaction in a fast  and growing enviroment. 

As we can see in this diagram each institution is represented as a different server. For this matter we used Docker to simulate each node of the Blockchain, and a final cluster of three databases, one for each node, in which we can see the consistency of the chain and blocks being created as transactions are made. Only one layer having to implement the front-end side, in this case RI (Registro Inmobiliario).

To come up with this idea we first interviewed users of the actual application, and created ideas based on this finds.

Conclusion

Used the classic Software Development Life Cycle (SDLC) model to design this application called Ivy. Identifying and attacking a problematic that is facing owners of real state in Dominican Republic, we in collaboration with RI (Registro Inmobiliario) will be presenting this new idea of a faster and safer way to manage the property of each individual.

We can assure the security and consistency of the data, no matter how many times a property has been traded. First to ensure the security we use Elliptic Curve Cryptography (ECC) implementation that assigns and validates each user key (As a matter of fact the U.S. National Security Agency recommends 256-bit ECC for protecting classified information up to the SECRET level, and 384-bit for TOP SECRET). And second is the consensus between the nodes to guarantee the validity and consistency of the data stored.
